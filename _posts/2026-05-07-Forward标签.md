#### 验证用户名与密码的流程：

```java
// message 用于存储报错原因
// flag 用于判断登录/注册是否成功
// 首先判断用户名是否为空，在判断输入框是否为有效输入
// 密码与确认密码同上
// 通过 equals() 方法，检测两次输入的密码是否相同
// 通过所有检测，登录/注册成功
String message = "";
Boolean flag = false;

if (username == null || username.trim().equals("")) {
    message = "用户名不能为空";
} else if (password == null || password.trim().equals("")) {
    message = "密码不能为空";
} else if (confirmedPassword == null || confirmedPassword.trim().equals("")) {
    message = "请再次输入密码";
} else if (!password.equals(confirmedPassword)) {
    message = "两次输入的密码不一致";
} else {
    message = "注册成功";
    flag = true;
}
```

#### `<jsp:forward>` 转发动作标签：

* 作用: 将请求从当前页面转发到目标页面
  `<jsp:forward page="目标页面路径"/>`
* page 属性：指定要转发到的目标页面路径

#### `<jsp:param>` 参数传递子标签：

* 作用: 向目标页面传递`额外`的请求参数
  `<jsp:param name="参数名" value="参数值"/>`
* name, value 属性：传递的参数名和参数值，其中值需要 `<%= %>`  来表示

#### 项目流程：

```bash
用户提交注册表单
    ↓
treatRegister.jsp 处理验证
    ↓
验证成功 (flag = true)
    ↓
执行 <jsp:forward> 转发
    ↓
传递参数: message="注册成功", username="用户输入的用户名"
    ↓
跳转到 successRegister.jsp
    ↓
successRegister.jsp 通过 request.getParameter() 接收参数
```

---

#### 总结：

* 通过 forward 标签，可以实现不改变地址栏跳转新页面

* 在当前收到的请求基础上，只需要传递额外的参数，实现共享一个请求对象
